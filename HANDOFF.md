# TojiWorks 작업 인수인계

이 문서는 다른 PC에서 Codex를 새로 실행해도 현재 작업을 이어갈 수 있게 남긴 작업 기록입니다.

## 서비스

- GitHub: https://github.com/dfc03300/toji
- Render: https://toji-hwux.onrender.com
- 로컬 실행: `npm start`
- 로컬 주소: http://127.0.0.1:5180

## 최근 반영 방향

- 서비스명은 `TojiWorks`.
- 왼쪽 사이드바는 미니멀한 텍스트 중심 UI.
- 버튼 순서: 파일 업로드, 파일 분석, 엑셀 다운로드, 작업관리.
- `작업관리` 아래 최근 작업 파일명은 최신순으로 5개만 표시.
- `더보기`는 사이드바를 펼치지 않고 작업관리 화면으로 이동.
- TojiWorks 로고 아이콘은 제거하고 텍스트만 유지.
- 사이드바 자체 스크롤은 만들지 않음.

## 엑셀 처리 핵심

- `processor.py`가 원본 엑셀을 헤더명 기준으로 읽는다.
- `거래시점 -> 시점`, 토지행의 `전체면적 -> 면적`, `지목`, `용도지역`을 매핑한다.
- `용도지역`은 약칭이 아니라 원문 그대로 넣는다.
- 공시지가는 realtyprice.kr API에서 캐시 없이 실시간 조회한다.
- API로 가져온 공시지가 텍스트는 붉은색으로 표시한다.

검증한 예:

- `성수동1가 656-938`
- 시점: `2026-04-08`
- 면적: `43`
- 지목: `대`
- 용도지역: `제2종일반주거지역`
- 개별공시지가: `11,040,000`

## 공시지가 조회 팝업

- 지번/도로명 라디오는 제거했다.
- 사용자가 주소나 지번을 입력하면 서버가 지번/도로명 후보를 함께 실시간 조회한다.
- 실제 realtyprice.kr API 결과가 있는 후보만 보여준다.
- 기준연도 입력은 필터 용도다. 추후 선택박스로 바꾸면 좋다.
- 원본 공시가격알리미 검증은 별도 버튼 링크가 아니라 팝업 오른쪽 슬라이드 패널로 열리게 설계했다.
- 모바일에서는 원본 패널이 하단 전체 패널처럼 열린다.

## Office365 편집

현재 M365 연동은 환경변수가 없으면 동작하지 않는다.

Render 환경변수:

- `MS_CLIENT_ID`
- `MS_CLIENT_SECRET`
- `MS_TENANT_ID=common`

Azure Redirect URI:

- 배포: `https://toji-hwux.onrender.com/auth/microsoft/callback`
- 로컬: `http://127.0.0.1:5180/auth/microsoft/callback`

## 배포

`render.yaml`에 `autoDeploy: true`를 명시했다.

배포가 반영되지 않으면 Render 대시보드에서:

- `Manual Deploy`
- `Deploy latest commit`

을 누르고 최신 커밋 SHA가 배포되는지 확인한다.
